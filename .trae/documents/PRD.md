## 1. 产品概述

纯前端浏览器本地运行的营养配餐工具，帮助用户（控糖人群、增肌健身者等）规划一日三餐，实时计算营养摄入、智能推荐餐单，无需任何后端服务。

- 解决用户手动计算每餐营养成分的繁琐问题，降低控糖/增肌饮食管理的认知负担
- 目标用户：需要控制血糖的家庭用户、需要增肌减脂的健身人群、关注营养均衡的普通用户
- 产品价值：纯本地运行，隐私安全；实时计算，所见即所得；智能配餐，降低决策成本

## 2. 核心功能

### 2.1 用户角色
无需角色区分，单用户使用。

### 2.2 功能模块
1. **目标设置区**：每日热量目标区间、三大营养素供能比目标、钠上限、餐次划分、忌口/过敏原排除
2. **食材库管理**：内置食材营养数据（每100g）、增删改、过敏原标注
3. **餐单编辑区**：按餐次（早/中/晚/加餐）添加食材和份量
4. **营养仪表盘**：当天总热量、各营养素总量、供能比环形图、达标进度条、超标提醒
5. **智能配餐引擎**：一键生成满足约束的建议餐单，策略独立可插拔
6. **数据持久化**：localStorage 本地存储、JSON 导入导出

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面（单页应用） | 顶部导航栏 | 应用标题、导入/导出按钮、重置按钮 |
| 主页面 | 目标设置面板 | 设置热量区间、三大营养素供能比（%）、钠上限（mg）、餐次启用、过敏原/忌口排除 |
| 主页面 | 食材库侧边栏 | 食材列表展示、搜索、新增、编辑、删除、过敏原标签 |
| 主页面 | 餐单编辑区 | 四个餐次卡片（早餐/午餐/晚餐/加餐），每个卡片内可添加食材、调整份量、删除食材 |
| 主页面 | 营养仪表盘 | 总热量数值卡片、蛋白质/脂肪/碳水供能比环形图、各营养素进度条、超标警示 |
| 主页面 | 智能配餐按钮区 | 一键智能配餐按钮、配餐结果提示（成功/失败原因） |

## 3. 核心流程

用户打开应用 → 查看/调整每日目标 → 浏览/编辑食材库 → 手动添加食材到各餐次或点击智能配餐 → 实时查看营养计算结果和图表 → 保存到本地或导出 JSON

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深翠绿 `#1a5c3a`（健康自然）+ 暖珊瑚 `#e07a5f`（警示超标）
- **辅助色**：米白 `#f4f1de`（背景）、暖灰 `#81b29a`（达标色）、土黄 `#f2cc8f`（接近达标）
- **按钮风格**：圆角 8px，主按钮深翠绿带渐变阴影，悬停微上浮
- **字体**：展示字体用 Lora（衬线，温润），正文字体用 DM Sans（无衬线，清晰易读）
- **布局风格**：卡片式布局，柔和阴影，大量留白，左侧食材库 + 中间餐单 + 右侧仪表盘三栏
- **图标风格**：lucide-react 线性图标，配食物 emoji 增加亲和力

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 主页面 | 目标设置面板 | 可折叠卡片，数字输入框+滑块组合，过敏原多选标签 |
| 主页面 | 食材库侧边栏 | 搜索框 + 食材列表项（名称+热量+过敏原标签+编辑删除按钮），底部悬浮新增按钮 |
| 主页面 | 餐单编辑区 | 四个餐次卡片横向排列，每个卡片带 emoji 图标、食材列表、份量输入、加减按钮、"+ 添加食材"按钮 |
| 主页面 | 营养仪表盘 | 大数字热量卡片居中，下方环形图（蛋白/脂肪/碳水三段），再下方垂直进度条组（热量、蛋白、脂肪、碳水、钠、纤维），超标项红色边框闪烁提示 |
| 主页面 | 智能配餐区 | 突出的主按钮，配餐结果提示条（成功绿色/失败红色），列出约束满足情况 |

### 4.3 响应式
桌面端三栏布局（食材库 25% + 餐单 45% + 仪表盘 30%），平板端两栏（食材库折叠为抽屉，餐单+仪表盘上下排列），移动端单列（所有模块堆叠，食材库为底部抽屉）。

### 4.4 动效
- 页面加载时各模块从下往上渐入（stagger 30ms）
- 食材添加到餐单时有飞入动画
- 营养数值变化时有数字滚动过渡
- 超标进度条有轻微呼吸红色光晕
- 环形图供能比变化时有平滑过渡
